# add-mc-bandwidth-render-mode

## 背景

MC带宽监控、滤波和控制输入本质上是按监控/控制周期锁存的离散值。阶梯线能表达硬件语义，但在多个PARTID同图对比时可读性较差，用户难以识别带宽趋势和不同QoS融合路径的差异。

## 目标

- 在控制总览的MC目标带/监控/实际图上增加显示模式选择。
- 默认使用插值曲线显示，便于看趋势和多PARTID对比。
- 保留硬件阶梯显示模式，便于检查周期锁存语义。
- 插值曲线必须保留采样点提示，并且只改变前端画法，不改变监控、滤波、控制输入或仿真结果。

## 非目标

- 不改变MC监控、滤波、QoS调整、BMIN/BMAX、CBusy或调度算法。
- 不新增仿真模式、数据模型或UI数据通路。
