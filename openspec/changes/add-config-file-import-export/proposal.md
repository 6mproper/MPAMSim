## 背景

当前控制台已有服务端预设和可编辑表单，但用户无法把当前配置保存为可分享文件，
也无法从文件恢复一次复杂实验配置。后续方案验证会频繁比较不同MPAM和流控参数，
需要可复现的本地配置包。

## 目标

- 在配置区提供“导出配置”和“导入配置”能力。
- 导出当前表单参数、PARTID配置、16线程激励和resctrl-like组为JSON文件。
- 导入同格式文件后恢复表单配置，并复用现有参数收集和表单填充路径。
- 导入格式必须带版本和类型标识；同时兼容只包含`parameters`对象的文件。

## 非目标

- 不在服务器端维护配置文件列表。
- 不新增数据库或用户账号。
- 不改变仿真配置schema和运行API。
